# 项目行为准则 (Project Rules)

你正在协助开发 **Gmail 智能清理工具 (Gmail Smart Cleaner)**。
这是一个基于 Google Agent Development Kit (ADK) 的 Web 应用，旨在帮助用户通过自然语言管理 Gmail 收件箱。

## 1. 项目核心原则
- **安全第一**：任何涉及用户数据的操作，都必须优先考虑安全性和隐私保护。
- **精准执行**：遵循“越细越精准”的原则，在不确定时主动请求澄清，不要猜测用户意图。
- **生产级质量**：产出的代码应具备良好的结构、错误处理和可读性，可直接用于部署。

## 2. 技术栈约束
- **Agent 框架**：必须使用 Google ADK (Python版)，ADK版本大于1.28，现在最新是2.0版本，尽量用最新的做。
- **大模型**：默认使用 Google Gemini。
- **工具集成**：通过 MCP (Model Context Protocol) 连接 Gmail，MCP我已配置好，你应该可以读取到。
- **后端**：将 ADK Agent 作为独立微服务部署。
- **前端**：可先用 ADK Web 调试，但最终需构建独立的 Web 应用（如 React/Vue）。
- **部署**：最终部署到 Google Cloud Run。

## 3. 安全与合规 (必须严格遵守)
- **认证**：必须使用 Google OAuth 2.0，用户密码不可见。
- **权限**：仅申请 `gmail.readonly` 和 `gmail.modify` 权限，**绝不申请 `gmail.send`**。
- **凭证**：所有 API Key 和敏感信息必须通过环境变量管理，禁止硬编码。
- **数据隐私**：邮件数据仅在用户会话中处理，不得存储到任何服务器或日志中。
- **操作确认**：执行“删除”等关键操作前，必须设计二次确认机制。

## 4. 功能实现要求
- **邮件读取**：默认读取最近7天的未读邮件，最多50封。
- **智能摘要**：调用 Gemini 生成不超过50字的中文摘要，并提取发件人和时间。
- **自动分类**：将邮件分为“需处理”、“可稍后看”、“直接删”三类。
- **批量操作**：提供“一键标记”、“一键归档”、“一键删除（需二次确认）”功能。
- **用户体验**：操作需有明确的加载状态和错误提示。
- 用户种类：多用户使用，只需登入账号即可使用。

## 5. 代码与风格规范
- **注释**：关键函数和复杂逻辑必须添加中文和英文注释。
- **命名**：变量和函数名使用英文，清晰表达其用途。
- **文档**：项目根目录需包含一个清晰的 `README.md`，说明项目背景、技术架构和启动步骤。
- **依赖**：每次添加新依赖时，需同时更新 `requirements.txt`。

## 6. 开发与交互方式
- **阶段式交付**：请将任务拆解为多个可交付的阶段（如：阶段一：搭建项目骨架；阶段二：实现Gmail连接；阶段三：开发核心功能...），并在完成每个阶段后向我汇报。
- **主动求助**：当遇到无法确定的技术选型或实现路径时，请主动列出选项并询问我的意见。
- **解释原则**：在给出代码前，请简要解释你的设计思路和关键决策点。
